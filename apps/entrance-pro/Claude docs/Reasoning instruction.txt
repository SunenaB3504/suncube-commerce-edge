CRITICAL CIRCULAR ARRANGEMENT RULES:

Generate 75 questions covering all number series types.

NUMBER SERIES TAXONOMY — Generate questions for ALL types:

TYPE 1: SIMPLE ARITHMETIC SERIES
  Rule: Each term increases/decreases by SAME number
  Example: 3, 7, 11, 15, 19, __, __
  Answer: +4 each time → 23, 27
  Easy — MAH CET staple — speedTarget: 20 seconds

TYPE 2: GEOMETRIC SERIES
  Rule: Each term multiplied/divided by SAME number
  Example: 2, 6, 18, 54, __, __
  Answer: ×3 each time → 162, 486
  Easy-Medium — both exams

TYPE 3: DIFFERENCE SERIES (CUET FAVOURITE)
  Rule: Differences between terms form their own series
  Example: 2, 6, 12, 20, 30, __, __
    // Exact example from user's framework
  Differences: 4, 6, 8, 10 → next difference = 12
  Answer: 30 + 12 = 42 → then 42 + 14 = 56
  This is the HIGHEST PRIORITY series type for CUET

TYPE 4: SQUARE/CUBE SERIES
  Square series: 1, 4, 9, 16, 25, __, __
    (n² series → 36, 49)
  Square + constant: 2, 5, 10, 17, 26, __
    (n² + 1 → 37)
  Cube series: 1, 8, 27, 64, __, __
    (n³ → 125)
  CUET loves n² ± k and n³ ± k patterns

TYPE 5: FIBONACCI-TYPE SERIES
  Rule: Each term = sum of two preceding terms
  Classic: 1, 1, 2, 3, 5, 8, 13, __, __
    (→ 21, 34)
  Modified: 1, 3, 4, 7, 11, 18, __, __
    (same rule, different start → 29, 47)

TYPE 6: MIXED OPERATION SERIES
  Rule: Operations ALTERNATE (+, ×, +, × etc.)
  Example: 3, 6, 10, 20, 24, 48, __, __
    (+3, ×2, +10, ×2... wait — verify pattern)
    Actually: ×2, +4, ×2, +4... → 52, 104
  MAH CET loves these — spot the alternating rule

TYPE 7: TWO-TIER DIFFERENCE SERIES (CUET HARD)
  Rule: Differences of differences form a series
  Example: 5, 7, 10, 14, 19, 25, __, __
    Differences: 2, 3, 4, 5, 6 → next = 7
    Answer: 25 + 7 = 32 → 32 + 8 = 40
  CUET staple — requires two levels of difference analysis

TYPE 8: PRIME NUMBER SERIES
  Example: 2, 3, 5, 7, 11, 13, __, __
    (consecutive primes → 17, 19)
  Mixed: 2, 5, 11, 17, 23, __, __
    (primes with gap: every other prime starting from 2)

TYPE 9: WRONG TERM SERIES (FIND THE ODD ONE)
  Instead of finding next term: find the WRONG term
  Example: 1, 4, 9, 16, 24, 36
    Wrong term: 24 (should be 25 = 5²)
  CUET uses this as a trap — trained students
  look for next term but question asks for wrong term

TYPE 10: NUMBER PAIR/TRIPLET SERIES (CUET ADVANCED)
  Pattern: (2, 4, 8), (3, 9, 27), (4, 16, ??)
    Rule: (n, n², n³) → answer: 64
  Teaches: Recognise n, n², n³ relationship

QUESTION FORMAT FOR EACH TYPE:

  Standard (find next term):
    "What comes next in the series: 2, 6, 12, 20, 30, ?"
    cognitiveType: "pattern-recognition"
    solution_steps must show:
      Step 1: Calculate differences
      Step 2: Identify pattern in differences
      Step 3: Apply to find answer

  Wrong term format:
    "One number in the series is WRONG. Find it:
    1, 4, 9, 16, 24, 36, 49"
    cognitiveType: "elimination"

  Fill the blank (CUET multi-blank):
    "2, ?, 12, ?, 30, 42"
    Find BOTH missing terms
    cognitiveType: "pattern-recognition"

DISTRIBUTION (75 questions):
  Type 1 (arithmetic): 10 questions
  Type 2 (geometric): 8 questions
  Type 3 (difference): 12 questions  ← highest priority
  Type 4 (square/cube): 10 questions
  Type 5 (Fibonacci): 6 questions
  Type 6 (mixed): 8 questions
  Type 7 (two-tier): 8 questions  ← CUET hard
  Type 8 (prime): 5 questions
  Type 9 (wrong term): 5 questions
  Type 10 (pairs/triplets): 3 questions

STUDY NOTES — 5 notes:
  Note 1: "First step ALWAYS — compute differences.
    Never guess the rule by inspection alone."
  Note 2: "Difference series — the 2, 6, 12, 20 type.
    Always check if differences themselves increase

  Note 3: "Square and cube — memorise 1³–15³ and

SYLLOGISM FUNDAMENTALS:

STATEMENT TYPES:

  Universal Negative: "No A is B" (A∩B=∅)
  Particular Positive: "Some A are B" (A∩B≠∅)
  Particular Negative: "Some A are not B" (A∩B≠A)

VALID CONCLUSION RULES (CUET logic):

  Rule 1: All + All → All
    "All A are B" + "All B are C" → "All A are C" ✓

  Rule 2: All + No → No
    "All A are B" + "No B is C" → "No A is C" ✓

  Rule 3: All + Some → Some
    "All A are B" + "Some B are C" → "Some A are C" ✗
    TRAP: This conclusion is NOT valid
    (Some B = C does not mean the A-part of B is C)

  Rule 4: Some + All → Some
    "Some A are B" + "All B are C" → "Some A are C" ✓

  Rule 5: Some + No → Some not
    "Some A are B" + "No B is C" → "Some A are not C" ✓

  Rule 6: Some + Some → No definite conclusion ✗
    "Some A are B" + "Some B are C" → NOTHING definite

  Rule 7: No + All → Some not (reverse)
    "No A is B" + "All B are C" → "Some C are not A" ✓

COMPLEMENTARY PAIRS (CUET TRAP):
  "All A are B" and "Some A are not B" = complementary
  Exactly one must be true — use this to eliminate
  "Either conclusion 1 or conclusion 2 follows" questions

VENN DIAGRAM METHOD vs FORMULA METHOD:
  MAH CET: Use Venn diagrams — draw circles, check visually
  CUET: Use formula rules — faster for 2-statement, required
    for 3-statement chains

FORMAT 2: Three statements (CUET hard)

    II. No B is C.
    III. Some C are D.
  Conclusions:
    I. No A is C.
    II. Some D are not B.
    III. Some C are not A.
  CUET tests 3-chain syllogisms — requires systematic approach

FORMAT 3: "Either...or" conclusion (CUET trap)
  When neither conclusion follows individually,

  "Either I or II follows" is the answer
  This is the #1 missed syllogism question type

FORMAT 4: Possibility cases (CUET advanced)

  Rule: Possibility is valid unless there's a

  for the statement to make sense.

  Rule for valid assumption:
    → Neither too broad (covers more than statement implies)
    → Nor too narrow (ignores key part of statement)
    → Must be necessary (without it, statement fails)
    → Must not be a direct restatement of statement

  Example:
    Statement: "Buy this medicine and get rid of cold."
    Assumption I: "The medicine is effective for cold."
    Assumption II: "Cold is a common ailment."
    Analysis:
      I: Valid — for the claim to make sense, the medicine
         must actually work. Implicit. Necessary.
      II: Invalid — whether cold is common or rare does
         not affect whether THIS medicine works.
    Answer: Only Assumption I is implicit.

  Common Traps:
    → Assumption too general: "People want to be healthy"
      — valid for health ads but too vague for specific claims
    → Assumption that negates premise: Always invalid
    → Future prediction as assumption: Usually invalid

  Conclusion must FOLLOW LOGICALLY from the statement.
  Not what is likely — what MUST be true if statement is true.

    Conclusion II: "No child in the class scored below 80%."

## ANTI-HALLUCINATION RULES

1. SERIES: Every answer must be verified by testing
   the derived rule against ALL given terms.
   Never publish a series rule that fails on any term.

2. SYLLOGISM: Never mark "All + Some → Some" as valid.
   This is the #1 hallucination in AI-generated logic.
   Rule: Some B are C + All A are B ≠ Some A are C
   (Unless all A are B AND some of those A-B overlap C)

3. CODING: Never assume the shift — always derive it.
   If "CAT = XZW" is given, compute C→X, A→Z, T→W
   and verify they follow the SAME rule before using.

4. ARRANGEMENTS: Every puzzle must be tested by the agent
   before writing questions. Place all N people, verify
   all constraints are satisfied, then write questions
   about that specific arrangement.

5. BLOOD RELATIONS: Never assume gender unless explicitly
   stated. "Child" is not "son". "Sibling" is not
   "brother". When gender is ambiguous — the answer
   may be "Cannot be determined".

6. DIRECTION: Shadow logic:
   Sun in East (morning) → shadow falls WEST.
   A person's shadow is BEHIND them.
   If shadow is West → sun is East → person faces East.
   Do NOT write questions where shadow logic is reversed.

7. CLOCK MIRROR: Formula is 11:60 - original time.
   Test: 3:00 → 11:60 - 3:00 = 8:60 = 9:00. Correct.
   Test: 6:00 → 11:60 - 6:00 = 5:60 = 6:00. Correct
   (6:00 is symmetric).
   Always verify before publishing.

8. VISUAL QUESTIONS: Every visual question must have
   a textDescription. Never publish a visual question
   with only a visualAssetRef and no text backup.
   The app must be usable even if images fail to load.

9. PUZZLE UNIQUENESS: Before writing any puzzle set,
   verify in your reasoning that the puzzle has
   EXACTLY ONE valid solution. If multiple solutions
   exist — add one more constraint and re-verify.
